# Template Evolution Reference

## Purpose
Document the evolution of templates based on emerging patterns, user feedback, and system learning.

## Template Lifecycle

### 1. Pattern Emergence
- **Detection**: Patterns identified through usage analysis
- **Validation**: Minimum 5 instances with consistent characteristics
- **Confidence**: Must exceed 75% similarity threshold

### 2. Template Candidate
- **Proposal**: Document pattern characteristics and use cases
- **Review**: Evaluate differentiation from existing templates
- **Prototype**: Create initial template structure

### 3. Template Development
- **Design**: Define folder structure and core files
- **Components**: Identify reusable elements
- **Commands**: Create specialized workflow commands
- **Testing**: Validate with real projects

### 4. Template Release
- **Documentation**: Complete README and usage guide
- **Integration**: Add to smart-bootstrap detection
- **Announcement**: Notify users of availability

### 5. Template Maturity
- **Monitoring**: Track usage and success rates
- **Refinement**: Iterate based on feedback
- **Optimization**: Improve efficiency and accuracy

### 6. Template Retirement
- **Obsolescence**: Technology or practice changes
- **Replacement**: Superseded by better template
- **Migration**: Guide for existing projects

## Evolution Triggers

### User-Driven Evolution
- **Feature Requests**: Direct user feedback
- **Pain Points**: Recurring issues or inefficiencies
- **New Use Cases**: Unanticipated applications

### Pattern-Driven Evolution
- **Frequency Threshold**: >5 similar projects
- **Efficiency Gains**: >30% time savings potential
- **Complexity Patterns**: Recurring complex setups

### Technology-Driven Evolution
- **New Technologies**: Emerging tools or frameworks
- **Best Practices**: Industry standard changes
- **Deprecations**: Obsolete technology replacement

## Template Combination Patterns

### Common Hybrids
| Primary | Secondary | Use Case | Handling |
|---------|-----------|----------|----------|
| Power Query | Documentation | BI solution with guides | Use primary with doc components |
| Research | Power Query | Data analysis research | Use Research with PQ tools |
| Base | Any | Unknown requirements | Start Base, evolve as needed |

### Hybrid Resolution Strategy
1. Identify primary domain (highest keyword match)
2. Note secondary characteristics
3. Use primary template as foundation
4. Add secondary template components as needed
5. Document hybrid nature in CLAUDE.md

## Template Quality Metrics

### Success Indicators
- **Setup Time**: <5 minutes for clear requirements
- **Template Accuracy**: >90% correct selection
- **User Modifications**: <20% of generated files
- **Task Completion Rate**: >80% without major changes

### Improvement Opportunities
| Metric | Current | Target | Improvement Method |
|--------|---------|--------|-------------------|
| Selection Confidence | Variable | >85% | Better pattern matching |
| Setup Completeness | 70% | >90% | More comprehensive templates |
| User Satisfaction | Unknown | >4/5 | Feedback collection needed |

## Version Control Strategy

### Versioning Scheme
- **Major (X.0.0)**: Structural changes, breaking updates
- **Minor (0.X.0)**: New features, non-breaking additions
- **Patch (0.0.X)**: Bug fixes, minor improvements

### Backward Compatibility
- Maintain compatibility within major versions
- Provide migration guides for major updates
- Support previous major version for 3 months

## Template Testing Framework

### Pre-Release Testing
1. **Unit Testing**: Individual component validation
2. **Integration Testing**: Full template generation
3. **User Testing**: Beta testing with volunteers
4. **Regression Testing**: Ensure existing features work

### Post-Release Monitoring
- Usage frequency tracking
- Error rate monitoring
- User feedback collection
- Performance metrics

## Template Deprecation Policy

### Deprecation Triggers
- No usage for 6 months
- Superseded by better alternative
- Technology becomes obsolete

### Deprecation Process
1. **Notice**: 3-month advance warning
2. **Migration Guide**: Provide transition path
3. **Support Period**: 3 months after deprecation
4. **Archive**: Maintain read-only reference
